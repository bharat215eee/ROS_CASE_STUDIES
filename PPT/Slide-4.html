1 Service Structure

ROS services are defined in .srv files. They consist of a Request and a Response, separated by three dashes (---).

  • Request

    The client sends a "mode" string (e.g., "square").

  • ---
  • Response

    Server returns success status and a message.

2 Setup Commands

$

cd ~/catkin_ws/src/case_study3_turtlesim_service

$

mkdir -p srv

$

gedit srv/RunMotion.srv &

# Verify after build:
$

rossrv show case_study3_turtlesim_service/RunMotion

srv/RunMotion.srv
1
string mode # "line", "circle", "square"
2
---
3
bool success
4
string message
UTF-8 ROS Service Definition

Important: The triple dash separator --- is mandatory. It tells the build system where the Request ends and the Response begins.
