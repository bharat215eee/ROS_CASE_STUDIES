Parameter Server

Case Study-3: Runtime Configuration

Step 6 of 8

Core Concept

The Parameter Server allows you to store and manipulate data (configuration parameters) accessible by all nodes. This enables tuning motion behavior at runtime without modifying code or restarting nodes.

Available Parameters

float

/motion/linear_speed

float

/motion/angular_speed

float

/motion/side_length

int

/motion/repetitions

float

/motion/line_duration

CLI Workflow

user@ros-machine: ~
1. Set Parameters

user@ros:~$ rosparam set /motion/linear_speed 1.5

user@ros:~$ rosparam set /motion/side_length 3.0

user@ros:~$ rosparam set /motion/repetitions 2

2. List All Parameters

user@ros:~$ rosparam list

/motion/angular_speed
/motion/circle_duration
/motion/linear_speed
/motion/repetitions
/motion/side_length
/rosdistro
/rosversion
...
3. Get Specific Value

user@ros:~$ rosparam get /motion/linear_speed

1.5
